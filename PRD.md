## 1. 문서 개요

### 1.1 프로젝트 명
**한국 노마드 시티** (Korean Nomad City)

### 1.2 문서 목적
본 문서는 한국 노마드 시티 웹 플랫폼의 **홈페이지** 개발을 위한 제품 요구사항을 정의합니다. 이 문서는 개발팀, 디자인팀, 마케팅팀이 동일한 비전을 공유하고 효율적으로 협업할 수 있도록 명확한 가이드라인을 제공합니다.

### 1.3 문서 범위
- ✅ **포함**: 홈페이지 UI/UX, 도시 카드 리스트, 필터/검색, 기본 정보 표시
- ❌ **제외**: 상세 페이지, 커뮤니티, 밋업, 회원 시스템, 관리자 페이지

### 1.4 주요 이해관계자
- **제품 책임자**: PM
- **개발팀**: Frontend/Backend Engineers
- **디자인팀**: UI/UX Designer
- **마케팅팀**: Growth Manager

## 2. 문제 정의

### 2.1 현재 상황
한국에서 디지털 노마드로 생활하고 싶은 사람들이 증가하고 있으나, **한국 도시별 생활 조건을 한눈에 비교하고 평가할 수 있는 플랫폼이 존재하지 않습니다.**

### 2.2 핵심 문제점

#### **문제 1: 정보 파편화**
- 도시별 생활비, 인터넷 속도, 카페 정보 등이 여러 사이트에 흩어져 있음
- 커뮤니티 후기는 블로그나 SNS에 산재
- 정보를 수집하는 데 많은 시간과 노력 소요

#### **문제 2: 신뢰할 수 있는 데이터 부재**
- 주관적인 블로그 후기에 의존
- 최신 정보 업데이트가 느림
- 객관적인 비교 지표 없음

#### **문제 3: 한국 특화 정보 부족**
- Nomads.com 같은 글로벌 플랫폼은 한국 도시 정보가 부족
- 한국 노마드 문화(카페 작업, 월정액, 코워킹 스페이스 등)를 반영하지 못함

### 2.3 사용자의 고민
> "부산에서 3개월 살면서 일하고 싶은데, 생활비가 얼마나 들까?"  
> "강릉은 인터넷이 빠른가? 작업하기 좋은 카페가 많나?"  
> "제주에서 다른 노마드들을 만날 수 있을까?"

```
┌─────────────────────────────────────────┐
│  👤 지수 (28세, 프리랜서 개발자)        │
├─────────────────────────────────────────┤
│  💼 직업: 풀스택 개발자 (원격 근무)     │
│  📍 거주지: 서울 → 지방 도시 탐색 중    │
│  💰 월 수입: 500만원                     │
│  🎯 목표: 워라밸 + 새로운 경험          │
├─────────────────────────────────────────┤
│  📱 기술 친화도: 높음                    │
│  🌐 SNS 활동: Instagram, 블로그 운영    │
│  ☕ 작업 환경: 주로 카페, 가끔 코워킹   │
├─────────────────────────────────────────┤
│  Pain Points:                            │
│  • 도시별 생활비 비교가 어려움          │
│  • 인터넷 속도 걱정                      │
│  • 커뮤니티 찾기 힘듦                    │
├─────────────────────────────────────────┤
│  Needs:                                  │
│  • 한눈에 보는 도시 비교                │
│  • 신뢰할 수 있는 데이터                │
│  • 실제 경험자들의 후기                 │
└─────────────────────────────────────────┘
```

### 3.2 사용자 특성

#### **인구통계적 특성**
- **연령**: 25-40세
- **성별**: 제한 없음 (남성 55%, 여성 45% 예상)
- **직업**: 개발자, 디자이너, 작가, 마케터, 컨설턴트
- **소득**: 월 300만원 이상

#### **행동적 특성**
- ✅ 컴퓨터 활용 능력 **상**
- ✅ SNS 활발 사용 (Instagram, 블로그, 유튜브)
- ✅ 온라인 커뮤니티 참여 경험
- ✅ 리뷰 작성/공유에 적극적
- ✅ 새로운 서비스 얼리어답터

#### **라이프스타일**
- 장소 독립적 업무 가능
- 1-6개월 단위 도시 이동
- 카페/코워킹 스페이스 작업 선호
- 로컬 경험 중시
- 일과 삶의 균형 추구

### 3.3 Secondary Persona

#### **기업 HR 담당자 "민지"**
- 워케이션 프로그램 기획
- 팀 빌딩 장소 선정
- 직원 복지 향상 목표

#### **디지털 노마드 예비자 "준호"**
- 회사 퇴사 준비 중
- 노마드 생활 정보 수집 단계
- 불안감 해소가 필요

## 4. 제안 해결책

### 4.1 솔루션 개요
**"한국 노마드 시티"는 대한민국 도시별 노마드 생활 조건을 한눈에 비교하고, 실사용자 평가를 기반으로 최적의 도시를 찾을 수 있는 플랫폼입니다.**

### 4.2 핵심 가치 제안 (Value Proposition)

```
╔═══════════════════════════════════════════════╗
║                                               ║
║  🎯 For 디지털 노마드를 꿈꾸는 사람들        ║
║                                               ║
║  😰 Who 한국 도시 정보를 찾기 어려워하는    ║
║                                               ║
║  🚀 Our Product 한국 노마드 시티는           ║
║                                               ║
║  ✨ That 도시별 생활비, 업무환경,            ║
║      커뮤니티 정보를 한곳에서 제공하는       ║
║                                               ║
║  🌟 Unlike Nomads.com (글로벌 중심) 또는     ║
║         블로그 (파편화된 정보)                ║
║                                               ║
║  💎 Our Product 한국 특화 데이터와           ║
║                실사용자 리뷰로 신뢰도 확보   ║
║                                               ║
╚═══════════════════════════════════════════════╝
```

### 4.3 차별화 요소

#### **1. 한국 특화 지표**
- 카페 월정액 평균 가격
- 코워킹 스페이스 밀집도
- 원룸/오피스텔 월세 (1인 기준)
- 배달 서비스 품질

#### **2. 실시간 데이터**
- 현재 날씨 & 대기질
- 실시간 인터넷 속도 테스트 결과
- 현재 체류 중인 노마드 수

#### **3. 직관적인 비교**
- 카드형 UI로 빠른 스캔 가능
- 다중 필터링 (예산, 환경, 목적)
- 시각적 지표 (별점, 게이지 바)

#### **4. 커뮤니티 기반 신뢰도**
- 실제 체류 인증 리뷰
- 도움됨 투표 시스템
- 검증된 사용자 배지

## 6. 기능 요구사항

### 6.1 기능 우선순위 (MoSCoW)

#### **Must Have (필수)**
- ✅ 도시 카드 리스트
- ✅ 기본 필터링 (예산, 환경, 목적)
- ✅ 정렬 기능 (평점순, 생활비순 등)
- ✅ 검색 기능
- ✅ 반응형 디자인

#### **Should Have (중요)**
- ✅ 지역별 탐색
- ✅ 실시간 통계 표시
- ✅ 밋업 미리보기
- ✅ 최신 리뷰 미리보기

#### **Could Have (선택)**
- ⭕ 다크모드
- ⭕ 애니메이션 효과
- ⭕ 소셜 공유 버튼

#### **Won't Have (제외)**
- ❌ 회원가입/로그인 (Phase 2)
- ❌ 리뷰 작성 (Phase 2)
- ❌ 커뮤니티 기능 (Phase 2)

### 6.2 홈페이지 기능 명세

#### **6.2.1 상단 네비게이션 바**

**기능 설명:**
사용자가 사이트 내 주요 섹션으로 이동할 수 있는 글로벌 네비게이션

**요구사항:**
```yaml
위치: 페이지 최상단 고정 (Sticky)
구성요소:
  - 로고: 클릭 시 홈으로 이동
  - 메뉴: [홈] [도시찾기] [커뮤니티] [밋업] [블로그]
  - 검색 아이콘: 클릭 시 검색 모달
  - 로그인/회원가입 버튼 (Phase 2)

스타일:
  - 배경: 반투명 화이트 (blur 효과)
  - 높이: 64px (데스크탑), 56px (모바일)
  - 그림자: subtle shadow

인터랙션:
  - 스크롤 시 배경 불투명도 증가
  - 현재 페이지 메뉴 하이라이트
  - 호버 시 색상 변화
```

#### **6.2.2 히어로 섹션**

**기능 설명:**
사용자에게 서비스의 가치를 즉시 전달하고 첫 액션을 유도

**요구사항:**
```yaml
구성요소:
  1. 메인 헤드라인:
     - 텍스트: "🌏 대한민국에서 일하기 가장 좋은 도시 찾기"
     - 폰트: Bold, 48px (데스크탑), 32px (모바일)
  
  2. 서브 헤드라인:
     - 텍스트: "디지털 노마드를 위한 완벽한 일과 삶의 균형을 찾아보세요"
     - 폰트: Regular, 20px (데스크탑), 16px (모바일)
  
  3. 검색 바:
     - Placeholder: "도시 이름 검색..."
     - 자동완성: Top 10 도시 추천
     - 엔터 키 또는 검색 버튼 클릭 시 검색 실행
  
  4. CTA 버튼:
     - 텍스트: "🚀 도시 탐색 시작하기"
     - 클릭 시 도시 리스트로 스크롤

배경:
  - 옵션 1: 한국 도시 풍경 그라디언트 오버레이
  - 옵션 2: 단색 배경 + 일러스트레이션

높이:
  - 데스크탑: 600px
  - 모바일: 500px
```

#### **6.2.3 실시간 통계**

**기능 설명:**
플랫폼의 활동성과 신뢰도를 시각적으로 보여줌

**요구사항:**
```yaml
표시 지표:
  1. 🏙️ 등록 도시 수: "47개"
  2. 👥 노마드 수: "1,234명"
  3. 💬 리뷰 수: "3,456개"
  4. 🎉 밋업 수: "89회"

레이아웃:
  - 4개 지표를 균등하게 배치
  - 각 지표는 아이콘 + 라벨 + 숫자

스타일:
  - 카드 형태, 약간의 elevation
  - 숫자는 카운팅 애니메이션 (optional)

업데이트:
  - 실시간 또는 매일 자정 업데이트
  - API에서 데이터 fetch
```

#### **6.2.4 인기 도시 TOP 10**

**기능 설명:**
가장 인기 있는 도시들을 카드 형태로 표시하여 빠른 탐색 유도

**요구사항:**
```yaml
데이터 소스:
  - 종합 평점 기준 상위 10개 도시
  - 또는 최근 30일 페이지뷰 기준

카드 당 표시 정보:
  1. 도시 대표 이미지 (16:9 비율)
  2. 배지: "인기" 또는 "떠오름"
  3. 도시명 + 행정구역
  4. 종합 평점 (⭐ 4.8)
  5. 현재 노마드 수 (👥 324명)
  6. 월 생활비 (💰 180만원)
  7. 평균 인터넷 속도 (📡 500 Mbps)
  8. 현재 날씨 (🌡️ 23°C ☀️)
  9. 주요 평가 지표 (업무환경, 생활비)
  10. 태그 3개 (#카페, #힐링, #자연)
  11. [보기] [❤️ 찜] 버튼

레이아웃:
  - 데스크탑: 3열 그리드
  - 태블릿: 2열 그리드
  - 모바일: 1열 리스트

인터랙션:
  - 카드 호버 시 살짝 상승 (elevation)
  - 이미지 호버 시 약간 확대
  - 찜하기 버튼 클릭 시 하트 채워짐 (로그인 필요 시 모달)
  - 카드 클릭 시 상세 페이지로 이동 (Phase 2)

하단 버튼:
  - [전체보기 →] 버튼
  - 클릭 시 전체 도시 리스트 페이지로 이동
```

#### **6.2.5 필터 및 검색 시스템**

**기능 설명:**
사용자가 조건에 맞는 도시를 빠르게 찾을 수 있도록 지원

**요구사항:**
```yaml
검색 기능:
  - 입력: 텍스트 입력창
  - 검색 대상: 도시명, 지역, 태그
  - 자동완성: 입력 시 실시간 추천
  - 최근 검색어: 로컬 스토리지에 저장 (최대 5개)

빠른 필터 (Quick Filters):
  1. 💰 예산:
     - [저예산] [중예산] [고예산]
     - 저: 150만원 미만
     - 중: 150-250만원
     - 고: 250만원 이상
  
  2. 🏖️ 환경:
     - [바다] [산] [도시] [자연]
  
  3. 🎯 목적:
     - [집중작업] [네트워킹] [워케이션]
  
  4. 🌡️ 계절:
     - [봄] [여름] [가을] [겨울]

정렬 옵션:
  - [종합 평점순]
  - [생활비 저렴한 순]
  - [인터넷 빠른 순]
  - [노마드 많은 순]
  - [최근 리뷰 많은 순]

UI:
  - 필터는 체크박스/버튼 형태
  - 다중 선택 가능
  - 선택된 필터는 하이라이트
  - 필터 개수 표시 (예: "3개 필터 적용됨")
  - [필터 초기화] 버튼

결과 표시:
  - "47개 도시 중 12개 발견"
  - 필터링된 결과를 즉시 카드 리스트에 반영
```

#### **6.2.6 지역별 탐색**

**기능 설명:**
한국 지역별로 도시를 탐색할 수 있는 빠른 네비게이션

**요구사항:**
```yaml
지역 카테고리:
  - 🏖️ 제주/섬
  - 🏙️ 수도권 (서울, 경기)
  - 🌊 강원 (강릉, 속초, 춘천 등)
  - 🏛️ 충청 (대전, 청주 등)
  - 🏞️ 전라 (전주, 여수 등)
  - 🏔️ 경상 (부산, 대구, 경주 등)

UI:
  - 큰 아이콘 버튼 형태
  - 6개 버튼을 2행 3열 배치 (데스크탑)
  - 2행 배치 (모바일)

인터랙션:
  - 클릭 시 해당 지역 도시만 필터링
  - 또는 지역별 페이지로 이동 (Phase 2)
```

#### **6.2.7 목적별 추천**

**기능 설명:**
사용자의 목적에 맞는 도시를 큐레이션하여 제시

**요구사항:**
```yaml
추천 카테고리:
  1. 💼 집중 작업
     - 설명: "조용하고 카페가 많은 도시"
     - 추천 도시: 강릉, 경주, 춘천
  
  2. 🎉 네트워킹
     - 설명: "활발한 커뮤니티와 밋업이 많은 곳"
     - 추천 도시: 서울, 부산, 제주
  
  3. 🏖️ 워케이션
     - 설명: "일과 휴양을 동시에!"
     - 추천 도시: 제주, 속초, 여수

레이아웃:
  - 3개 카드를 가로 배치
  - 각 카드: 아이콘 + 제목 + 설명 + 추천 도시 + CTA

CTA:
  - [둘러보기 →] 버튼
  - 클릭 시 해당 목적으로 필터링된 도시 리스트
```

#### **6.2.8 커뮤니티 미리보기**

**기능 설명:**

**요구사항:**
```yaml
구성 (2열 레이아웃):

왼쪽: 📅 다가오는 밋업

  - 각 밋업 정보:
    • 도시 아이콘 + 이름
    • 일시 (예: 11/20 (수) 오후 7시)
    • 참여 인원 (예: 12/20명)
    • [참여하기] 버튼
  - 하단: [모든 밋업 보기 →]

오른쪽: 👋 새로 가입한 노마드
  - 최근 가입자 프로필 사진 12개 (4x3 그리드)
  - 설명 텍스트: "개발자, 디자이너, 마케터 등 다양한 분야의 노마드들이 함께하고 있습니다!"
  - 하단: [커뮤니티 가입 →]

데이터 소스:
  - 밋업: 최신순 3개 (API)
  - 신규 회원: 최근 가입순 12명 (API)
```

## 11. 부록

### 11.1 용어 정의

```yaml
디지털 노마드 (Digital Nomad):
  - 장소에 구애받지 않고 원격으로 일하며 여러 도시를 여행하는 사람

워케이션 (Workation):
  - Work + Vacation의 합성어. 휴가지에서 일하며 여가를 즐기는 것

코워킹 스페이스 (Coworking Space):
  - 여러 사람이 함께 사용하는 공유 작업 공간

MAU (Monthly Active Users):
  - 월간 활성 사용자 수. 한 달 동안 서비스를 1회 이상 이용한 사용자

DAU (Daily Active Users):
  - 일일 활성 사용자 수

MVP (Minimum Viable Product):
  - 최소 기능 제품. 핵심 기능만 구현한 초기 버전
```
