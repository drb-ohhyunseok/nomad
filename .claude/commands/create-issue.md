# 깃허브 이슈 자동 생성기

입력된 내용을 분석하여 체계적이고 완성도 높은 깃허브 이슈를 생성합니다.

## 📝 이슈 내용
```
$ARGUMENT
```

## 🎯 실행 프로세스

### 1단계: 이슈 분석 및 분류

입력된 내용을 분석하여 다음을 결정합니다:

**이슈 타입 분류:**
- 🐛 **Bug**: 버그 리포트, 오류, 예상치 못한 동작
- ✨ **Feature**: 새로운 기능 요청, 개선 사항
- 📚 **Documentation**: 문서 개선, README 업데이트
- 🔧 **Refactor**: 코드 리팩토링, 구조 개선
- 🎨 **Design**: UI/UX 개선, 디자인 변경
- ⚡ **Performance**: 성능 최적화
- 🧪 **Test**: 테스트 추가 또는 개선
- 🏗️ **Chore**: 빌드, 설정, 의존성 관리

**우선순위 평가:**
- 🔴 **Critical**: 즉시 해결 필요 (프로덕션 중단, 보안 이슈)
- 🟠 **High**: 빠른 해결 필요 (주요 기능 영향)
- 🟡 **Medium**: 일반적인 우선순위
- 🟢 **Low**: 낮은 우선순위 (개선 사항)

### 2단계: 이슈 제목 생성

**제목 형식 규칙:**
- 타입 이모지 + 간결한 설명 (50자 이내)
- 명령형 동사로 시작 (Fix, Add, Update, Improve 등)
- 구체적이고 검색 가능한 키워드 포함

**예시:**
- `🐛 Fix: 로그인 시 세션 만료 오류`
- `✨ Feature: 도시 상세페이지 구현`
- `⚡ Performance: 이미지 로딩 최적화`

### 3단계: 이슈 본문 구조화

선택된 이슈 타입에 따라 적절한 템플릿을 사용합니다:

### 4단계: 메타데이터 설정

**라벨 제안:**
- 이슈 타입에 따른 기본 라벨
- 우선순위 라벨
- 관련 영역 라벨 (frontend, backend, database 등)

**마일스톤 고려:**
- 프로젝트의 현재 진행 상황 확인
- 적절한 마일스톤이 있으면 제안

**담당자 제안:**
- 이슈 내용과 관련된 영역의 담당자 제안 (알고 있는 경우)

### 5단계: 깃허브 이슈 생성

다음 정보를 사용하여 이슈를 생성합니다:

1. **분석 결과 요약 제시**
   - 분류된 이슈 타입
   - 제안된 제목
   - 우선순위
   - 제안 라벨

2. **사용자 확인 요청**
   - 제목이나 내용 수정 필요 여부 확인
   - 추가 정보 필요 여부 확인

3. **gh CLI 명령어 생성**
   ```bash
   gh issue create \
     --title "제목" \
     --body "본문 내용" \
     --label "label1,label2" \
     --assignee "@me"
   ```

4. **이슈 생성 실행**
   - 사용자 승인 후 gh CLI로 이슈 생성
   - 생성된 이슈 URL 반환

## 🔍 추가 분석 항목

### 프로젝트 컨텍스트 확인
- SPEC.md나 README.md 확인하여 프로젝트 구조 파악
- 기존 이슈들의 패턴 확인 (가능한 경우)
- package.json에서 사용 중인 기술 스택 확인

### 자동 연결
- 관련된 다른 이슈가 있는지 검색
- PR과 연결 필요 여부 확인
- 프로젝트 보드에 자동 추가 제안

## 🎨 출력 형식

최종 출력은 다음 구조를 따릅니다:

```markdown
# 🎯 이슈 생성 준비 완료

## 📊 분석 결과
- **타입**: [이슈 타입]
- **우선순위**: [우선순위]
- **예상 작업 시간**: [시간 추정]

## 📝 제안 이슈

### 제목
[생성된 제목]

### 본문
[생성된 본문]

### 메타데이터
- **Labels**: label1, label2, label3
- **Assignees**: [제안된 담당자]
- **Milestone**: [해당하는 경우]

## 🚀 생성 명령어

다음 명령어로 이슈를 생성할 수 있습니다:

[gh CLI 명령어]

## ❓ 다음 단계
1. 이슈 내용 확인 및 수정 필요 여부 알려주세요
2. 승인하시면 바로 이슈를 생성하겠습니다
3. 수정이 필요하면 어느 부분을 어떻게 수정할지 알려주세요
```

## 💡 사용 예시

```bash
# 버그 리포트
/create-issue 로그인 페이지에서 이메일 입력 시 한글이 입력되지 않습니다

# 기능 요청
/create-issue 도시 상세페이지 구현 - 도시 클릭 시 상세 정보를 보여주는 페이지 필요

# 문서 개선
/create-issue README에 프로젝트 설치 및 실행 방법 추가

# 리팩토링
/create-issue CityCard 컴포넌트의 중복 로직 제거 및 최적화
```

## ⚙️ 설정 옵션

다음과 같은 추가 옵션을 지원합니다:

- `--dry-run`: 이슈를 생성하지 않고 미리보기만 표시
- `--label`: 추가할 라벨 지정
- `--priority`: 우선순위 강제 지정 (critical/high/medium/low)
- `--assign`: 담당자 직접 지정

## 🔒 보안 및 검증

- 민감한 정보 (API 키, 비밀번호 등) 포함 여부 자동 검사
- 이슈 내용이 너무 짧거나 불명확한 경우 추가 정보 요청
- gh CLI 인증 상태 확인

## 📋 실행 지침

1. **입력 분석**: $ARGUMENT 내용을 면밀히 분석
2. **타입 자동 분류**: 키워드 기반 이슈 타입 결정
3. **템플릿 적용**: 적절한 템플릿으로 본문 구조화
4. **메타데이터 생성**: 라벨, 마일스톤 등 자동 제안
5. **사용자 확인**: 생성 전 최종 확인 요청
6. **이슈 생성**: gh CLI로 이슈 생성 및 URL 반환

이제 입력된 내용으로 깃허브 이슈를 생성하겠습니다.
